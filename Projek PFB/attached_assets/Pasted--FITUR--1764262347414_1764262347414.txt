======================================================================
=                     FITUR OVERTIME (LEMBUR) WAJIB ADA              =
======================================================================

Tambahkan modul dan tabel khusus untuk fitur lembur (overtime). 
Sistem harus mendukung:

1. Pengajuan Overtime (oleh pekerja)
2. Persetujuan Overtime (oleh admin)
3. Tracking jam lembur otomatis
4. Perhitungan durasi overtime
5. Riwayat Overtime

======================================================================
=                         DATABASE OVERTIME                           =
======================================================================

Tambahkan tabel baru di MySQL bernama: overtime

Struktur tabel:
- id (INT, AUTO_INCREMENT)
- employee_id (INT)
- date (DATE)
- start_time (DATETIME)
- end_time (DATETIME)
- duration (INT / dalam menit)
- reason (TEXT)
- status (PENDING / APPROVED / REJECTED)
- approved_by (INT, nullable)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)

======================================================================
=                     HALAMAN OVERTIME YANG HARUS ADA                =
======================================================================

1. overtime.php  (Halaman utama lembur untuk pekerja)
   Berisi:
   - Form pengajuan lembur:
     - tanggal lembur
     - jam mulai
     - jam selesai
     - alasan lembur
   - List status pengajuan lembur:
     - Pending
     - Approved
     - Rejected

2. overtime_admin.php  (Panel admin approval lembur)
   Harus berisi:
   - Tabel daftar pengajuan lembur
   - Filter berdasarkan:
     - tanggal
     - nama pekerja
     - status
   - Tombol:
     - Approve
     - Reject
   - Otomatis hitung durasi lembur dari start_time & end_time

======================================================================
=                 LOGIKA OVERTIME DI BACKEND PHP                     =
======================================================================

1. Pekerja mengajukan lembur:
   - Disimpan ke tabel overtime
   - Status awal: PENDING

2. Admin memproses lembur:
   - Approve → status jadi APPROVED
   - Reject → status jadi REJECTED
   - approved_by diisi user admin

3. Durasi lembur dihitung otomatis:
   duration = (end_time - start_time) dalam menit

4. Validasi wajib:
   - Jam selesai harus > jam mulai
   - Tidak boleh overlap dengan jam kerja reguler
   - Tidak boleh mengajukan lembur di hari yang sama lebih dari 1 kali

5. Riwayat lembur:
   - Setiap pekerja dapat melihat daftar lemburnya
   - Admin bisa melihat semua lembur pekerja

======================================================================
=                        DASHBOARD OVERTIME                           =
======================================================================

Tambahkan ke dashboard.php:
- Card “Total Lembur Bulan Ini”
- Card “Lembur Pending Approval”
- Tabel ringkasan lembur terbaru
- Grafik total jam lembur per minggu/bulan (opsional)
